## ⚠️ 评测协议

**时序留一法**：每个用户按时间戳排序物品，**前 80% 作为历史（建画像），后 20% 作为评测目标**。历史与评测完全隔离，零泄漏。所有方法使用同一套划分，公平对比。

### 核心对比

| 对比维度 | 结果 |
|---------|------|
| 时序 vs 非时序最优 | SASRecF (0.851) vs DeepFM+Feat (0.707) = **+20.4% GAUC** |
| 特征增强（SASRec→SASRecF） | 0.825 → 0.851 = **+3.2% GAUC**，**+6.1% Recall@10** |
| 特征增强（BERT4Rec→BERT4RecF） | 0.822 → 0.796 = **-3.1%**（MASK 冲突，待重训验证 fix） |
| 特征增强（DeepFM→DeepFM+Feat） | 0.676 → 0.707 = **+4.6%** |
| SASRecF vs BERT4Rec | 0.851 vs 0.822，SASRecF 领先 **+3.6%** |

### 关键发现

1. 🎯 **SASRecF 是新的 SOTA**：GAUC 0.8509，首次突破 0.85 大关
2. 📈 **特征增强在 SASRec 上效果显著**：+3.2% GAUC、+6.1% Recall@10，远超 DeepFM 上的 +4.6%
3. 🔄 **单向注意力 + 特征天然契合**：无 MASK token 冲突，FeatureSeqEmbLayer 直接注入
4. ⚠️ **BERT4RecF 掉点确认是 MASK bug**：0.822→0.796 (-3.1%)，fix 后重训预期恢复甚至反超
5. 🚀 **时序建模 + 特征增强是最优组合**：SASRecF (0.851) vs DeepFM+Feat (0.707) = +20.4%，证明两者缺一不可

## 🧪 SASRecP 三项创新（已实现，待评测）

| 模块 | 配置开关 | 原理 | 预期收益 |
|------|---------|------|---------|
| **RoPE** | `use_rope` | 旋转位置编码，attention 依赖相对位置差 | 长短序列用户统一建模 |
| **Time Interval Bias** | `use_time_bias` | 真实时间间隔作为 attention bias | 捕捉用户兴趣漂移速度 |
| **Contrastive Loss** | `use_contra_loss` | 同 category 物品 embedding 更近 | 改善物品表征，提升跨用户泛化 |

每个模块独立可控，支持 2³ = 8 组消融实验。额外开关：
- `time_bias_method`: `log` / `linear`
- `contra_weighting`: `binary` / `hierarchical`（L1/L2/L3 层级权重）
- `contra_lambda`: 对比 loss 权重（默认 0.1）
