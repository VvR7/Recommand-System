## ⚠️ 评测协议

**时序留一法**：每个用户按时间戳排序物品，**前 80% 作为历史（建画像），后 20% 作为评测目标**。历史与评测完全隔离，零泄漏。所有方法使用同一套划分，公平对比。

> T = Time Interval Bias, C = Feature-aware Contrastive Loss

### 消融分析（完整，含单模块独立验证）

| 实验 | GAUC | vs SASRecF | 累计增益 |
|------|------|-----------|---------|
| SASRec (ID-only) | 0.8249 | — | — |
| + item cat features (=SASRecF) | 0.8509 | — | +3.2% |
| SASRecF + TimeBias only | 0.8712 | +2.4% | — |
| SASRecF + Contrastive only | **0.8702** | **+2.3%** | — |
| SASRecF + TimeBias + Contrastive | 0.8839 | +3.9% | **+7.1%** (over SASRec) |

> **关键发现**：TimeBias (+2.4%) 和 Contrastive (+2.3%) 单独效果几乎持平，但两者存在部分功能重叠——联合使用 +3.9% < 2.4% + 2.3%。即便如此，两者叠加仍带来了可观的互补增益，证明它们从不同维度提升了推荐质量（TimeBias 利用时间信号，Contrastive 利用语义信号）。

### ❌ RoPE 被证明无效

| 实验 | GAUC | 说明 |
|------|------|------|
| SASRecF (基线) | 0.8509 | — |
| SASRecF + RoPE | 0.8029 | **-4.8%**，严重退化 |

> 原因：left-padded 序列中，绝对位置 embedding 编码了"距序列末尾多远"的关键语义。RoPE 丢掉绝对位置信号后，模型丧失了序列尾部感知能力——这对 next-item prediction 是致命的。RoPE 不适合 left-padded 序列推荐场景。

### BERT4RecF（特征增强双向，已确认设计限制）

| 方法 | GAUC | Recall@10 | 说明 |
|------|------|-----------|------|
| BERT4RecF | 0.7963 | 0.4790 | Cloze masking 与特征注入结构性冲突，掉点 -3.1% |
| BERT4Rec (ID-only) | 0.8217 | 0.5274 | 基线 |

### 核心对比

| 对比维度 | 结果 |
|---------|------|
| 最佳模型 vs 传统最优 | SASRecF+T+C (0.884) vs DeepFM+Feat (0.707) = **+25.0% GAUC** |
| 特征增强贡献 | SASRec → SASRecF: +3.2% |
| TimeBias 单独贡献 | SASRecF → SASRecF+T: +2.4% |
| Contrastive 单独贡献 | SASRecF → SASRecF+C: +2.3% |
| 联合贡献 | SASRecF → SASRecF+T+C: +3.9%（两者互补，重叠部分约 0.8%） |
| 总提升 | SASRec → 最佳: 0.825 → 0.884 = **+7.1%** |
| BERT4Rec vs BERT4RecF | 0.822 → 0.796 = -3.1%（设计限制） |

### 关键发现

1. 🎯 **SASRecF+T+C 是最终 SOTA**：GAUC 0.8839，Recall@20 突破 0.81
2. ⏱ **Time Interval Bias (+2.4%) 和 Contrastive Loss (+2.3%) 单独效果几乎持平**，两者从不同维度提升推荐质量
3. 🤝 **联合使用互补增益 +3.9%**，虽存在部分重叠（~0.8%）但叠加效果显著
4. ❌ **RoPE 不适合 left-padded 序列推荐**：掉点 -4.8%，论文可作为负样本发现
5. ⚠️ **BERT4Rec 不适合特征注入**：Cloze masking 与特征注入结构性冲突

## 🧪 有效创新模块

| 模块 | 配置开关 | 原理 | 效果 |
|------|---------|------|------|
| **Time Interval Bias** | `use_time_bias` | 真实时间间隔作为 attention bias | **+2.4%** ✅ |
| **Contrastive Loss** | `use_contra_loss` | 同 category 物品 embedding 更近 | **+1.5%** ✅ |
| ~~RoPE~~ | ~~`use_rope`~~ | ~~旋转位置编码~~ | **-4.8%** ❌ 已放弃 |

每个模块独立可控，可自由组合。额外参数：
- `time_bias_method`: `log` / `linear`
- `contra_weighting`: `binary` / `hierarchical`（L1/L2/L3 层级权重）
- `contra_lambda`: 对比 loss 权重（默认 0.1）

## 数据预处理 & 特征工程

### 交互数据（`dataset/kuairec/kuairec.inter`）
- 来源：`big_matrix.csv`，经 k-core 过滤（user≥5, item≥5），wr∈[0.7,2.0]
- 预处理脚本：`id_only_baseline/scripts/preprocess.py`
- 结果：5,765 用户，3,310 物品，4.17M 交互

### 用户特征（`dataset/kuairec/kuairec.user`）
- 来源：`KuaiRec/KuaiRec/KuaiRec2.0/data/user_features.csv`（7,176 用户）
- 预处理脚本：**`preprocess_features.py`**
- 选用 12 个 token 字段
- ⚠️ 跨用户评测不可用（仅 DeepFM+Feat 训练时辅助学习）

### 物品特征（`dataset/kuairec/kuairec.item`）
- 来源：`KuaiRec/kuairec_caption_category.csv`（三级类别层级）
- 预处理脚本：**`preprocess_features.py`**
- 选用 3 个 token 字段：L1(39) / L2(109) / L3(156)
- ✅ 训练+评测均可用（物品空间统一）
