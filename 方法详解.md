# 推荐系统方法详解 — KuaiRec 数据集

> 本文档详细描述数据处理流程、实验/评测数据划分，以及所有 12 个方法的数学模型和数据流向。

---

## 1. 数据处理全流程

### 1.1 原始数据

KuaiRec 2.0 数据集（快手短视频推荐），包含：

```
原始文件：
├── big_matrix.csv          # 训练交互矩阵 (12.5M 行, 7,176 用户 × 10,728 物品)
├── small_matrix.csv        # 测试交互矩阵 (4.7M 行, 1,411 用户 × 3,327 物品)
├── user_features.csv       # 用户特征 (7,176 行)
├── kuairec_caption_category.csv  # 物品三级类别 (10,728 行)
└── item_daily_features.csv # 物品逐日特征 (343K 行)
```

**关键特性**：
- big_matrix 与 small_matrix 的 **用户完全无交集**（训练用户 5,765，测试用户 1,129）
- **物品空间统一**：以 small_matrix 的 3,310 个物品（实际为 3,327）为准，大矩阵只保留这些物品的交互
- 每个交互有 `watch_ratio`（播放比例 = 实际播放 / 视频时长），作为偏好强度信号

### 1.2 数据预处理

#### Step 1: 生成交互文件 (`kuairec.inter`)

```
输入: big_matrix.csv
处理:
  1. k-core 过滤: 保留出现 ≥5 次的用户和物品
  2. watch_ratio 截断: 仅保留 wr ∈ [0.7, 2.0] 的正例
     (wr < 0.7 的低质量交互作为后续评测的困难负例)
  3. 物品空间统一: 仅保留 small_matrix 中存在的 3,310 个物品
  4. 格式转换: user_id \t video_id \t timestamp → RecBole .inter 格式
输出: dataset/kuairec/kuairec.inter (4.17M 交互, 5,765 用户, 3,310 物品)
```

#### Step 2: 生成特征文件 (`kuairec.user` + `kuairec.item`)

```
输入:
  - user_features.csv (用户画像)
  - kuairec_caption_category.csv (物品三级类别)

处理 user 特征:
  选取 12 个 token 字段: user_active_degree, is_live_streamer,
  is_video_author, follow_user_num_range, fans_user_num_range,
  friend_user_num_range, register_days_range, onehot_feat0~3
  → 输出 kuairec.user (7,176 用户, TSV 格式, token:token:... header)

处理 item 特征:
  选取 3 个 token 字段:
  - first_level_category_id  (L1, 39 类)
  - second_level_category_id (L2, 109 类)
  - third_level_category_id  (L3, 156 类)
  过滤至 3,310 个物品 → 输出 kuairec.item
```

### 1.3 训练 vs 评测数据

| 维度 | 训练集 | 测试集 |
|------|--------|--------|
| 数据来源 | big_matrix 过滤后 | small_matrix (1,129 用户) |
| 用户数 | 5,765 | 1,129 |
| 交互数 | ~4.17M | ~2.09M 正例 |
| 用户重叠 | — | **0（完全不重叠）** |
| 物品重叠 | — | **100%（3,310 物品共享）** |

> **这是跨用户泛化评测**：模型在 5,765 个用户上训练，在完全不同的 1,129 个用户上评测。用户特征（性别、活跃度等）不可用于评测——模型只能用物品的 ID 和类别特征来做推荐。

### 1.4 评测数据构造（时序留一法 80/20）

```
对测试集每个用户:
  1. 取出所有 wr ≥ 0.7 的正例
  2. 按 timestamp 升序排列
  3. 前 80% → 历史序列 (构建用户画像)
  4. 后 20% → 评测目标 (不在历史中，零泄漏)
  5. 困难负例: 该用户 wr < 0.3 的物品，固定采样 99 个 (seed=42)
  6. 无时间戳的物品 (small_matrix 实验采集) → 追加至历史

对每个评测目标:
  构造候选池 {1 positive + 99 hard negatives}
  → 模型对 100 个候选打分 → 计算 GAUC / Recall@K / NDCG@K
  → 用户内取均值 → 跨用户取整体均值
```

**关键统计**：1,129 用户 × 平均 359 评测目标 = **404,944 评测样本**（vs Leave-One-Out 仅 1,129）

## 2. 模型方法详解

### 2.1 符号约定

| 符号 | 含义 |
|------|------|
| $\mathcal{U}$ | 用户集合 |
| $\mathcal{I}$ | 物品集合，$\vert\mathcal{I}\vert=3,310$ |
| $\mathbf{e}_i \in \mathbb{R}^d$ | 物品 $i$ 的 embedding，$d=256$ |
| $s_u = (i_1, i_2, \dots, i_L)$ | 用户 $u$ 的交互序列，按时间排序，左侧 zero-pad 至长度 $L=200$ |
| $\mathbf{E} \in \mathbb{R}^{\vert\mathcal{I}\vert \times d}$ | 物品 embedding 矩阵 |
| $\mathbf{p}_k \in \mathbb{R}^d$ | 位置 $k$ 的绝对位置 embedding |
| $\mathcal{L}$ | 损失函数 |

---

### 2.2 传统方法（非时序）

#### 2.2.1 ItemKNN（基于物品的协同过滤）

**核心思想**：物品之间的相似度用共现统计量衡量。

**训练**：
1. 构造用户-物品交互矩阵 $\mathbf{R} \in \{0,1\}^{\vert\mathcal{U}\vert \times \vert\mathcal{I}\vert}$
2. 计算物品-物品余弦相似度矩阵 $\mathbf{W} \in \mathbb{R}^{\vert\mathcal{I}\vert \times \vert\mathcal{I}\vert}$

$$w_{ij} = \frac{\mathbf{R}_{:,i}^\top \mathbf{R}_{:,j}}{\Vert\mathbf{R}_{:,i}\Vert \cdot \Vert\mathbf{R}_{:,j}\Vert}$$

**评测**（给定用户历史 $\mathcal{H}_u \subset \mathcal{I}$）：

$$s(u, i) = \frac{1}{\vert\mathcal{H}_u\vert} \sum_{j \in \mathcal{H}_u} w_{ji}$$

即对历史物品的相似度取均值，作为候选物品的得分。

**数据流**：用户历史 ID 列表 → 查表取相似度行 → 均值聚合 → [3,310] 得分向量

---

#### 2.2.2 BPR（贝叶斯个性化排序）

**核心思想**：矩阵分解 + 成对排序损失。将每个用户 $u$ 表示为隐向量 $\mathbf{u} \in \mathbb{R}^d$，每个物品 $i$ 表示为 $\mathbf{i} \in \mathbb{R}^d$。

**模型**：

$$s(u, i) = \mathbf{u}^\top \mathbf{i}$$

**训练损失**（BPR 成对损失）：

$$\mathcal{L}_{\text{BPR}} = -\frac{1}{\vert\mathcal{D}\vert} \sum_{(u,i^+,i^-) \in \mathcal{D}} \log \sigma\big(s(u,i^+) - s(u,i^-)\big)$$

其中 $\mathcal{D}$ 是从训练集采样的三元组：用户 $u$、正例 $i^+$（交互过的物品）、负例 $i^-$（未交互物品）。

**评测**：用户在训练中学习到的 embedding $\mathbf{u}$ 固定后，与所有物品 embedding 做内积得到排名。但在跨用户场景中，评测时无 $\mathbf{u}$（测试用户未见过），实际使用历史物品 embedding 的均值作为虚拟用户表示：

$$\hat{\mathbf{u}} = \frac{1}{\vert\mathcal{H}_u\vert} \sum_{i \in \mathcal{H}_u} \mathbf{e}_i, \quad s(u, j) = \hat{\mathbf{u}}^\top \mathbf{e}_j$$

**数据流**：历史物品 ID → embedding 查表 → 均值池化 → dot product → [3,310] 得分

---

#### 2.2.3 NeuMF（神经矩阵分解）

**核心思想**：GMF（广义矩阵分解）+ MLP 双路融合。

**模型结构**：

$$\begin{aligned}
\phi^{\text{GMF}}(u,i) &= \mathbf{p}_u^G \odot \mathbf{q}_i^G \in \mathbb{R}^d \\
\phi^{\text{MLP}}(u,i) &= \text{MLP}\big([\mathbf{p}_u^M; \mathbf{q}_i^M]\big) \in \mathbb{R}^d \\
\hat{y}_{ui} &= \sigma\left(\mathbf{w}^\top [\phi^{\text{GMF}}; \phi^{\text{MLP}}] + b\right)
\end{aligned}$$

其中 $\text{MLP}$ 结构为 $d \to 256 \to 128 \to 64 \to d$（逐层减半），激活函数为 ReLU。

**训练损失**：BCE（二元交叉熵）

$$\mathcal{L} = -\frac{1}{\vert\mathcal{D}\vert} \sum_{(u,i,y) \in \mathcal{D}} \big[y \log \hat{y}_{ui} + (1-y) \log(1-\hat{y}_{ui})\big]$$

**评测**：同 BPR，用户侧表示用历史物品 embedding 均值替代。

**数据流**：历史物品 embedding 均值 → 作为虚拟用户向量 → GMF 和 MLP 双路前向 → concat → 输出层 → [3,310] 得分

---

#### 2.2.4 DeepFM

**核心思想**：FM（因子分解机）捕获二阶特征交互 + Deep MLP 捕获高阶交互。

**ID-only 版本**：

输入特征仅为 `user_id` 和 `video_id` 两个 token 字段。每个字段有独立的 embedding 表。

**FM 部分**（二阶交互）：

$$y_{\text{FM}} = \sum_{i=1}^{m} \sum_{j=i+1}^{m} \langle \mathbf{v}_i, \mathbf{v}_j \rangle = \frac{1}{2} \sum_{f=1}^{d} \left[\left(\sum_{i=1}^{m} v_{if}\right)^2 - \sum_{i=1}^{m} v_{if}^2\right]$$

其中 $m$ 为特征字段数（ID-only 中 $m=2$），$\mathbf{v}_i \in \mathbb{R}^d$ 为第 $i$ 个字段的 embedding。

**Deep 部分**：

$$y_{\text{Deep}} = \text{MLP}\big([\mathbf{v}_1; \mathbf{v}_2; \dots; \mathbf{v}_m]\big)$$

**最终输出**：

$$\hat{y} = \sigma\Big(\underbrace{\sum_i w_i x_i}_{\text{一阶}} + y_{\text{FM}} + y_{\text{Deep}}\Big)$$

**评测**：用户 ID 用历史物品 embedding 的均值替代，物品 ID 用候选物品 embedding。

**数据流**：特征字段 → embedding 查表 → FM 交互 + MLP concat → 输出层 → [3,310] 得分

---

#### 2.2.5 DeepFM+Feat（特征增强版）

在 ID-only 基础上增加用户特征（12 个 token 字段）和物品特征（3 个 category token 字段）。

**特征字段构成**（共 17 个字段）：

| 类型 | 字段 | 数量 |
|------|------|------|
| 用户 ID | `user_id` | 1 |
| 物品 ID | `video_id` | 1 |
| 用户特征 | `user_active_degree`, `onehot_feat0~3`, ... | 12 |
| 物品特征 | `first_level_category_id`, `second_level_category_id`, `third_level_category_id` | 3 |

FM 部分的 pairwise 交互覆盖了所有跨类型特征组合（如 user_degree × item_category）。

**评测时**：用户侧特征不可用（测试用户未在训练中出现），用户特征字段用 **零 embedding** 替代。物品特征可用（物品空间统一）。

---

### 2.3 序列模型

所有序列模型共享以下基础结构：

- **输入**：用户交互序列 $s_u = (i_1, \dots, i_L)$，长度 $L=200$，左侧 zero-pad
- **左填充语义**：真实序列靠右对齐，position 0 = padding，position 199 = 最后一次交互
- **Embedding 维度**：$d=256$
- **Transformer 配置**：$n_{\text{layers}}=2$，$n_{\text{heads}}=4$，$d_{\text{ff}}=1024$
- **Dropout**：hidden_dropout=0.5，attn_dropout=0.5

---

#### 2.3.1 SASRec（自注意力序列推荐）

> Kang & McAuley, ICDM 2018

**核心思想**：使用单向（因果）Transformer 建模序列中的物品转移关系。

**模型结构**：

```
输入序列 s_u = (i_1, ..., i_L)
   │
   ├─ Item Embedding:    E_item(s_u)           → [B, L, d]
   ├─ Position Embedding: E_pos(0,1,...,L-1)   → [B, L, d]
   │
   ├─ 相加: x = E_item + E_pos                → [B, L, d]
   ├─ LayerNorm + Dropout
   │
   └─ TransformerEncoder × 2 layers:
        ├─ MultiHeadAttention (causal mask)
        │    Q,K,V = Linear(x)
        │    scores = softmax(QK^T/√d_k + M)   (M: 左下三角 = 0, 右上 = -∞)
        │    out = scores · V
        ├─ LayerNorm(x + Attention(x))
        ├─ FFN(x) = Linear(ReLU(Linear(x)))     (inner_size=1024)
        └─ LayerNorm(x + FFN(x))
        输出: h_u = 最后一个位置 (L-1) 的输出 → [B, d]
```

**因果注意力掩码** $\mathbf{M}$：

$$M_{ij} = \begin{cases} 0 & i \geq j \\ -\infty & i < j \end{cases}$$

确保位置 $i$ 只能 attend 到 $\leq i$ 的位置（不能看到未来）。

**训练**（BPR 损失）：

$$\mathcal{L}_{\text{BPR}} = -\sum_{(u,i^+,i^-)} \log \sigma\big(\mathbf{h}_u^\top \mathbf{e}_{i^+} - \mathbf{h}_u^\top \mathbf{e}_{i^-}\big)$$

**数据流**：序列 [B, L] → Item Emb + Pos Emb → LayerNorm + Dropout → 2×Transformer (causal) → gather_indexes(last_pos) → [B, d] → dot product with item embeddings → [B, |I|]

---

#### 2.3.2 SASRecF（特征增强 SASRec）

在 SASRec 基础上，为每个物品拼接其类别特征 embedding。

**关键改动**：

```
SASRec:
  x = E_item(s_u) + E_pos                     → 单一来源

SASRecF:
  item_emb   = E_item(s_u)                    → [B, L, d]
  feat_emb   = FeatureSeqEmbLayer(s_u)        → [B, L, 3×d]
               # 3 个类别特征字段 (L1, L2, L3)，每个 d 维
  concat     = [item_emb; feat_emb]           → [B, L, 4×d]
  x          = Linear_{4d→d}(concat) + E_pos  → [B, L, d]
```

`FeatureSeqEmbLayer` 为序列中每个物品位置查找其类别特征的 embedding，输出所有特征 embedding 的拼接。`concat_layer` 是线性映射 $\mathbb{R}^{4d} \to \mathbb{R}^d$。

**为什么比 ID-only 好**：
- 物品 ID embedding 只捕获协同过滤信号
- 类别特征提供语义信息（如"这个物品是音乐类"），对冷门物品有帮助
- 测试集物品与训练集完全相同 → 类别特征在训练和评测时均可用

**数据流**：
```
sequence [B,L] → Item Emb [B,L,d]
              → FeatureSeqEmbLayer:
                   L1 Emb [B,L,d] + L2 Emb [B,L,d] + L3 Emb [B,L,d]
                   → concat [B,L,3d]
              → [Item; Features] → Linear → [B,L,d]
              → + Pos Emb → LayerNorm → 2×Transformer → last_pos → [B,d]
              → dot(·, E_item) → scores [B,|I|]
```

---

#### 2.3.3 SASRecF+T（+ 时间间隔注意力偏置）★ 创新 1

在 SASRecF 的基础上，利用真实时间戳计算物品间的时间间隔，将其作为注意力偏置注入 Multi-Head Attention。

**核心动机**：序列中相邻物品的时间间隔包含重要信息——用户连续快速消费的内容通常话题相关，而间隔很久的物品之间关联较弱。

**时间偏置计算**：

给定序列的真实时间戳 $\mathbf{t} = (t_1, \dots, t_L)$（Unix 时间戳）：

$$\Delta_{ij} = \vert t_i - t_j \vert \quad \text{(pairwise 时间差)}$$

$$\mathbf{B}_{ij} = -w \cdot \log(1 + \Delta_{ij})$$

其中 $w > 0$ 是可学习参数（初始化为 0.01）。负号意味着：时间越近 → $B_{ij}$ 越接近 0 → attention score 惩罚越小 → 注意力权重越高。

**注入方式**（修改 MultiHeadAttention）：

$$\text{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}} + \underbrace{\mathbf{B}}_{\text{time bias}} + \mathbf{M}\right) \mathbf{V}$$

其中 $\mathbf{B} \in \mathbb{R}^{B \times 1 \times L \times L}$ 在 batch 维度广播到所有 head。

**训练时**：时间戳从 RecBole interaction 中获取（`timestamp_list` 字段）
**评测时**：历史序列的时间戳从测试数据中提取，传递到 scorer 的 `compute_time_bias()`

**数据流**：
```
同 SASRecF，额外:
  timestamps [B,L] → compute_time_bias()
    → pairwise |t_i - t_j| → log1p → -w * Δ → [B,1,L,L]
    → 注入 attention: scores += time_bias (before softmax)
```

---

#### 2.3.4 SASRecF+C（+ 特征感知对比学习）★ 创新 2

在 SASRecF 的主损失之上增加 InfoNCE 对比学习辅助损失（完整公式见 [2.5.1 损失函数对照](#251-损失函数对照)）。

**核心动机**：主损失（BPR）只关心物品 ID 级别的相对排序，忽略了物品之间的语义关系。对比损失直接作用于物品 embedding 矩阵，让同一类别的物品在归一化空间中更近。

**层级权重**（Hierarchical）：

$$w_{ij} = \begin{cases}
1.0 & i,j \text{ 同 L3 类别（最细粒度，156 类）} \\
0.5 & \text{同 L2 但不同 L3（109 类）} \\
0.2 & \text{同 L1 但不同 L2（39 类）} \\
0   & \text{无共同类别}
\end{cases}$$

**总损失**：$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{BPR}} + 0.1 \cdot \mathcal{L}_{\text{InfoNCE}}$，其中 `calculate_loss` 返回 `(main_loss, contra_loss)` tuple，RecBole trainer 自动求和。

**数据流**：
```
主路径:   同 SASRecF → BPR loss
辅助路径: batch 中独特物品 → 查表取 L1/L2/L3 → embedding 归一化
          → 层级加权 mask → InfoNCE loss
          → return (main_loss, contra_loss)  # trainer 自动求和
```

---

#### 2.3.5 SASRecF+T+C（联合模型）★ 最终 SOTA

同时启用 Time Interval Bias 和 Contrastive Loss。两者从不同维度增强推荐质量：

- **TimeBias** 利用时间信号（物品间的时间距离），作用于 attention 权重
- **Contrastive** 利用语义信号（物品间的类别关系），作用于 embedding 空间

**总损失**：$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{BPR}} + 0.1 \cdot \mathcal{L}_{\text{InfoNCE}}$（同 SASRecF+C）

**性能栈**（GAUC）：

```
SASRec        0.8249  基线
  → + item features    +3.2%   (=SASRecF 0.8509)
  → + TimeBias         +2.4%   (=SASRecF+T 0.8712)
  → + Contrastive      +1.5%   (=SASRecF+T+C 0.8839)
  —————————————————————————————————
  累计提升: +7.1%
```

两个模块从不同维度提升推荐质量：
- **TimeBias** 利用**时间信号**：物品间的时间距离
- **Contrastive** 利用**语义信号**：物品间的类别关系

联合使用 +3.9%（vs 2.4%+2.3%=4.7%），重叠约 0.8%，表明存在一定功能重叠但总体互补。

### 2.4 对比方法（BERT4Rec 系列）

#### 2.4.1 BERT4Rec

> Sun et al., CIKM 2019

**核心思想**：双向 Transformer + Cloze 任务。随机 mask 输入序列中 20% 的位置，用双向上下文预测被 mask 的物品。

**与 SASRec 的关键区别**：

| 维度 | SASRec | BERT4Rec |
|------|--------|----------|
| 注意力方向 | 单向（因果） | 双向 |
| 训练任务 | Next-item prediction | Cloze（填空） |
| Mask 策略 | 无需 mask（causal mask 自然阻止未来信息） | 随机 mask 20% 位置 |
| 输出 | 取最后位置 | 取 mask 位置并接输出层 |
| 损失函数 | BPR（成对排序） | CE（交叉熵，多分类） |

**模型结构**：

```
输入序列 s_u = (i_1, ..., i_L)
   │
   ├─ 随机 mask 20% 位置: s_u^mask
   │   被 mask 的位置替换为特殊 token [MASK]
   │
   ├─ Item Embedding + Position Embedding → x  [B, L, d]
   ├─ LayerNorm + Dropout
   │
   └─ TransformerEncoder × 2 layers (bidirectional attention)
        # 双向 = 无因果 mask，所有位置互相可见
        输出: [B, L, d]
        → 取 mask 位置 → [B, mask_num, d]
        → dot(W_item) + bias → logits [B, mask_num, |I|]
```

**训练损失**（交叉熵）：

$$\mathcal{L}_{\text{CE}} = -\frac{1}{\vert\mathcal{M}\vert} \sum_{(u,p) \in \mathcal{M}} \log \frac{\exp(\mathbf{h}_{up}^\top \mathbf{e}_p)}{\sum_{j \in \mathcal{I}} \exp(\mathbf{h}_{up}^\top \mathbf{e}_j)}$$

其中 $\mathcal{M}$ 为所有被 mask 的位置集合。

**评测**：Bert4Rec scorer 不使用 mask——直接将完整历史输入模型，对所有位置输出取均值池化。

---

#### 2.4.2 BERT4RecF（特征增强版，❌ 掉点）

在 BERT4Rec 基础上加入物品类别特征（同 SASRecF 的 FeatureSeqEmbLayer）。

**掉点原因（-3.1% GAUC）**：

```
BERT4RecF 训练时:
  序列: [真实, [MASK], 真实, [MASK], 真实, [MASK], ...]
                  ↑ 20% 位置是 MASK token
  MASK 位置的特征: 置零 (无法获取真实物品的类别)
  → 模型学到的模式: "有特征的上下文 → 预测 MASK 位置的物品"
  → 训练-推理分布不一致

BERT4RecF 推理时:
  序列: [真实, 真实, 真实, ..., 真实]  (无 MASK token)
  但模型在训练时很少见到"全部位置都有特征"的输入
  → 泛化到完整序列时表现下降
```

这是 Cloze masking 与特征注入的结构性冲突——MASK token 使得 20% 的训练样本缺少特征信息，而推理时无此问题但模型未充分适应。

#### 2.5.1 损失函数对照

| 方法 | 损失函数 | 公式 | 训练范式 |
|------|---------|------|---------|
| **BPR** | BPR | $\mathcal{L} = -\frac{1}{B}\sum \log\sigma(s^+ - s^-)$ | 成对排序 |
| **NeuMF** | BCE | $\mathcal{L} = -\frac{1}{B}\sum [y\log\hat{y} + (1-y)\log(1-\hat{y})]$ | 逐点分类 |
| **ItemKNN** | — | 无训练，直接计算物品相似度 | 非参数 |
| **DeepFM** / **DeepFM+Feat** | BCE | $\mathcal{L} = -\frac{1}{B}\sum [y\log\hat{y} + (1-y)\log(1-\hat{y})]$ | 逐点分类 |
| **SASRec** / **SASRecF** / **SASRecF+T** | BPR | $\mathcal{L} = -\frac{1}{B}\sum \log\sigma(\mathbf{h}_u \cdot (\mathbf{e}_{i^+} - \mathbf{e}_{i^-}))$ | 成对排序 |
| **SASRecF+C** / **SASRecF+T+C** | BPR + InfoNCE | $\mathcal{L} = \mathcal{L}_{\text{BPR}} + \lambda \cdot \mathcal{L}_{\text{InfoNCE}}$ | 成对排序 + 对比学习 |
| **BERT4Rec** / **BERT4RecF** | CE | $\mathcal{L} = -\frac{1}{\vert\mathcal{M}\vert}\sum_{(u,p)}\log\frac{e^{\mathbf{h}_{up}^\top \mathbf{e}_p}}{\sum_j e^{\mathbf{h}_{up}^\top \mathbf{e}_j}}$ | 多分类 |

> ItemKNN 无训练损失——直接从交互矩阵计算物品-物品余弦相似度矩阵 $\mathbf{W}$，评测时对历史物品相似度取均值。

#### 2.5.2 RecBole 实现细节

**BPRLoss 源码**（来自 `recbole.model.loss`）：

$$\boxed{\mathcal{L}_{\text{BPR}} = -\frac{1}{B} \sum_{i=1}^{B} \log\big(\gamma + \sigma(s_i^+ - s_i^-)\big)}$$

其中 $\gamma = 10^{-10}$ 防止 $\log(0)$。每个 batch 采样 $B$ 个三元组 $(u, i^+, i^-)$。

**InfoNCE 对比损失**（SASRecP 自定义）：

$$\boxed{\mathcal{L}_{\text{InfoNCE}} = -\frac{\lambda}{\vert\mathcal{P}\vert} \sum_{i: \exists j, w_{ij}>0} \log\frac{\sum_{j\neq i} w_{ij} \cdot e^{\tilde{\mathbf{e}}_i^\top \tilde{\mathbf{e}}_j / \tau}}{\sum_{k\neq i} e^{\tilde{\mathbf{e}}_i^\top \tilde{\mathbf{e}}_k / \tau}}}$$

其中 $\tau=0.07$, $\lambda=0.1$，$w_{ij}$ 为层级权重。

**Tuple 返回自动求和**：

```python
# SASRecP.calculate_loss() 返回
if self.use_contra_loss:
    return main_loss, contra_loss  # trainer 自动: total = main_loss + contra_loss
return main_loss
```

RecBole trainer 检测到 `calculate_loss` 返回 tuple 时，自动对 tuple 内所有元素求和作为最终训练损失。无需手动 `main_loss + contra_loss`。

**Cross-Entropy Loss**（BERT4Rec）：

$$\boxed{\mathcal{L}_{\text{CE}} = -\frac{1}{\vert\mathcal{M}\vert} \sum_{(u,p) \in \mathcal{M}} \log \frac{\exp(\mathbf{h}_{up}^\top \mathbf{e}_p + b_p)}{\sum_{j \in \mathcal{I}} \exp(\mathbf{h}_{up}^\top \mathbf{e}_j + b_j)}}$$

其中 $\mathbf{b} \in \mathbb{R}^{\vert\mathcal{I}\vert}$ 是物品级别的可学习偏置（`output_bias`），$\mathcal{M}$ 为所有被 mask 的位置。

## 3. 关键实现细节

### 3.1 序列 Padding 策略

所有序列模型使用 **left-padding**（左侧补零）：

```
短序列 (L=3):     [0, 0, ..., 0, i_1, i_2, i_3]
长序列 (L>200):   [i_{n-199}, ..., i_n]  (截断取最后 200)
```

位置编码从 0 到 199，真实交互占据高索引位置。**Position 199 始终是最后一次交互**——这个绝对位置信号对 next-item prediction 至关重要，这也是 RoPE 失败的根本原因。
